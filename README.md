# my-web-project
<어떤 기술 스택을 왜 선택했는지>
  프론트엔드 : 아무것도 모르는 상황에서 찾아봤을 때 기본기와 보안 원리를 공부하기 좋고 별도의 빌드 과정이 없어서 배포가 매우 빠르고 가볍고 브라우저가 코드를 어떻게 해석하는지 생으로 볼 수 있어 학습 효과가 크다고 해서 프론트엔드는 HTML/CSS/JS를 선택했다.
  백엔드 : python을 사용해봤고 코드가 짧고 구조가 명확하다고 해서 백엔드는 python + Flask 조합을 선택했다.
  DB : SQLite가 파일 하나짜리라 설치 없이 바로 시작 가능하다고 하고 SQL을 직접 손으로 작성하는게 나중에 SQL Injection이 왜 터지는지 이해할 때 큰 도움이 되는 경험이라고 해서 SQLite로 골랐다.
  프록시 : Nginx (리버스 프록시)를 사용했다.
  
<전체 구조>
  전체 구조는 (프론트엔드: HTML/CSS/JS, 백엔드: Python + Flask, DB: SQLite, 프록시: Nginx) 이다

<배포과정>
  먼저 python과 flask 를 설치한 후 폴더를 만들고 vs code와 터미널을 이용해서 브라우저에서 hello world가 뜨도록 했다.
  그다음 본격적으로 회원가입/로그인 기능을 만들기 시작했다.
  폴더 안 코드를 교체하고 '회원가입 성공!' 을 응답하게 하는 작업을 했다.
  로그인 API도 마찬가지로 '로그인 성공!' 이 뜨게 했다.
  새로운 폴더를 만들고 터미널에 코드를 작성해서 회원가입, 로그인 폼이 뜨게 했다.
  그다음 폴더 안 코드를 교체하고 터미널에 코드를 작성해서 프로필 체이지를 만들었다.
  이제 배포를 하기 위해 AWS에 가입해서 EC2서버를 만들었다.
  키 페어 이름은 my-web-key로 했다.
  PowerShell을 열고 코드를 작성해서 AWS 서버에 접속했다.
  코드를 넣고 Flask 서버를 실행하고 Nginx를 활성화하고 폴더와 HTML 파일들을 만들었다.
  그다음 가비아에 가입하고 도메인을 구매하고 DNS설정을 통해 레코드를 추가해 AWS서버 IP와 연결했다.
  그다음 HTTPS인증서를 발급하고 Certbot으로 HTTPS 적용 완료했다.
  마지막으로 서버가 재부팅해도 자동으로 켜지게 하는 코드를 넣고 서비스 등록했다.

<막혔던 부분>
  - Windows PowerShell에서 curl 명령어가 달라서 Invoke-WebRequest를 사용했다.
  - Flaskrk 127.0.0.1에만 바인딩되어 있어서 0.0.0.0으로 변경했다.
  - Nginx sites-enabled에 없는 파일이 참조되어 있어서 삭제 후 변경했다.
